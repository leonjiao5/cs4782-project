--- dora_layers ---
W = m * (W0 + BA) / ||W0 + BA||_c
- m is trainable (per-column magnitude vector)
- ||.||_c is column-wise L2 norm
- detach the norm in the denominator -- gradient trick from sec 4.3
- init: m = ||W0||_c, A ~ kaiming, B = 0  (so initial W = W0, no perturbation)
- forward needs to handle batched x; watch for shape bugs

--- model ---
- primary candidate: Llama (proposal commits to Llama, 7B / 8B band)
    - Llama-3.1-8B-Instruct  (default, gated -- need HF access)
    - Llama-3.2-3B-Instruct  (smaller, easier on T4 without 4-bit)
- alternates worth a look if Llama is a pain:
    - Qwen2.5-Math-7B-Instruct  (math-pretrained; usually strongest on AIME OOTB)
    - DeepSeek-R1-Distill-Qwen-7B  (reasoning-distilled, strong on AIME)
    - mistralai/Mistral-7B-Instruct-v0.3  (ungated 7B fallback)
- compute: Colab supports T4 (16GB), A100 (40GB), and H100 -- pick path per GPU:
    - T4         -> fp16 + 4-bit base via bitsandbytes
    - A100/H100  -> bf16, may not need quantization
- default targets: q_proj, k_proj, v_proj, o_proj
- maybe also gate_proj / up_proj / down_proj later
- freeze everything except DoRA params (and lm_head?)
- baselines: peft LoRA (paper comparison) + peft DoRA (sanity check on our impl)

--- data ---
- AIME 2024: Maxwell-Jia/AIME_2024  (30 rows = AIME I + II, fields: ID, Problem, Solution, Answer)
- AIME 2025: MathArena/aime_2025    (30 rows, fields: problem_idx, problem, answer, problem_type)
- only need problem + answer for eval -- drop the rest
- normalize on load: 2024 has capitalized fields, 2025 is lowercase
- answers are int64 in both; cast to str for comparison (AIME answers are 0-999)
- 2024 ID prefix encodes contest ("2024-I-x" vs "2024-II-x"); 2025 has no I/II label
- training corpora (locked):
    - MATH-lighteval                                  -- DigitalLearningGmbH/MATH-lighteval (~7.5K)
    - NuminaMath-CoT (sub-sample)                     -- AI-MO/NuminaMath-CoT
    - OpenMathInstruct-2 (deferred for train_scale)   -- nvidia/OpenMathInstruct-2
  dropped: HARP (no HF mirror), Omni-MATH (test-only), math-datasets-100k (no reasoning traces)

  are kept as training signal (do NOT drop the aops_forum / amc_aime sources wholesale)
- prompt format: chat template (likely) vs raw -- depends on base model

- code/data_clean/ is the data prep package (sources.py, leakage.py, build.py)
- HF datasets cache (~/.cache/huggingface/datasets) avoids re-downloads across tier builds
- run: python -m code.data_clean.build --tier {train_light,train,train_scale}
- train_light : DONE (7,498 rows, math-lighteval only; 2 dropped for missing \boxed)
- train       : DONE (27,498 rows = 7,498 lighteval + 20,000 NuminaMath sub-sample)
                NuminaMath sub-sample includes aops_forum, amc_aime, olympiads, etc.
                stage 2-4 leakage filter ran; 0 AIME 2024/2025 matches in the sub-sample
                (verified: filter catches verbatim AIME injection; no row scores Jaccard>=0.2)
- train_scale : TODO (OpenMathInstruct-2 download + filter pending)
- every output row: solution contains \boxed{...}, answer == extracted boxed -- enforced in sources.py
- code/data.py:extract_boxed_answer is the canonical extractor, used both at data prep and at eval
- audit log: results/logs/leakage_drops.jsonl  (empty for current build; safety net only)
- NuminaMath default sub-sample: 20K, seed=42 (tune later)

--- train ---
- AdamW, lr ~2e-4, cosine schedule, ~3% warmup
- precision: bf16 on A100/H100, fp16 + 4-bit base on T4
- gradient checkpointing on
- save every N steps; eval AIME every M steps

--- eval ---
- AIME answers are integers 0-999
- extract via \boxed{...}
- two settings: greedy + n-sample majority (n=16 or 64)
- pass@k for completeness

--- experiments ---
- main: DoRA(r) vs LoRA(r) at matched param budget on AIME 2024 + 2025
- secondary: DoRA(r/2) vs LoRA(r) parameter-efficiency claim (paper Table 1 / Fig 2)
- sanity: our custom DoRA vs peft's DoRA at same config -- should be within noise
